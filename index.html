BPMN that executes everywhere

Goals of this Project

  • Execute business processes everywhere
  • Provide visibility into any business flow
  • Improve communication about requirement changes
  • Use built-in services for common tasks
  • Write your own tasks using TypeScript

Philosophy

Custom software is often hard to adapt to requirement changes. Business processes not only give you the flexibility to adapt. They favor communication between technical and functional departments creating a better understanding on both sides.

The process engine treats business processes as both: the documentation and the blueprint for your software. Even in complex scenarios this minimizes the code you have to implement and thereby enhances productivity and stability of your solutions.

Key Features

Platform Independent

Run the software in your browser or in any Node.js environment - the same code where possible, the specific code where needed.

Database Independent

No matter if you use a relational, graph or object database or even a key/value-store - the process engine knows how to deal with it.

Dynamic UI

Use a dynamically generated UI to handle user tasks. Integrate your own UI into the process engine by using one of many client APIs.

Type Safety and Intellisense / TypeScript

The whole software stack is written in TypeScript - this gives you the confidence of type safety and a rich development experience by providing IntelliSense.

Identity and Access Management

The process engine provides a built-in rights management based on claims. Attach required claims to every property or method by using simple conventions.

Validation

Generate JSON Schemas from all of your TypeScript code to validate data and parameters at runtime.

Modularity

Thanks to the extremely modular codebase and IoC architecture you can replace EVERY aspect of the application if needed.

HTTP REST API

Expose any entities of your data layer in a feature-rich HTTP REST API. That way other software can utilize the data layer abstraction and the claims-based rights management of the process engine.

Process Integration

Invoke any TypeScript code in your whole software stack directly from within your business process. Use built-in services for common tasks such as sending e-mails, generating PDFs or calling other REST APIs.
